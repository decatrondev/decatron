# 10 - Internationalization (i18n): GachaVerse Migration

**Created:** 2026-04-04
**Project:** GachaVerse Migration
**Purpose:** Define the i18n structure for the migrated gacha system

---

## Current State

GachaVerse has **NO internationalization**. All user-facing text is hardcoded in Spanish throughout the application — bot responses, UI labels, error messages, and overlay text are all written directly in Spanish strings within the source code.

---

## Migration i18n Plan

The migrated gacha module will support three languages from day one:

| Language | Code | Priority | Notes |
|----------|------|----------|-------|
| Spanish | ES | Primary | Current user base language |
| English | EN | Secondary | International audience |
| Portuguese | PT | Tertiary | Brazilian Twitch audience |

Language selection follows Decatron's per-channel setting — each streamer configures their preferred language, and all gacha responses in that channel use that language.

## Implementation Details

### Decatron MessagesService Integration

Bot command responses use the existing `MessagesService` with language keys:

```csharp
// Example usage in GachaCommandModule:
var message = _messagesService.Get(
    channel.Language,           // per-channel language setting
    "gacha_cmd.pull_result",    // translation key
    new {                       // interpolation parameters
        user = participant.TwitchUsername,
        item = pulledItem.Name,
        rarity = _messagesService.Get(channel.Language, $"gacha.rarity.{pulledItem.Rarity.ToLower()}")
    }
);
```

### React i18n Integration

UI translations use Decatron's existing React i18n infrastructure:

```tsx
// Example usage in GachaConfig component:
import { useTranslation } from 'react-i18next';

const GachaConfig = () => {
    const { t } = useTranslation();

    return (
        <h1>{t('gacha.title')}</h1>
        <button>{t('gacha.btn.save')}</button>
    );
};
```

### Language Detection

Language is determined per channel, following the same logic as the rest of Decatron:

1. Check channel's configured language setting
2. Fall back to Decatron instance default language
3. Final fallback: Spanish (ES)

No per-viewer language override — all viewers in a channel see the same language. This is consistent with how Twitch bots operate (one language per chat room).

## Translation Workflow

1. **Development:** All new user-facing strings use translation keys (never hardcoded text)
2. **Primary language (ES):** Written during development
3. **Secondary languages (EN, PT):** Added in Phase 8 of migration
4. **Review:** Native speakers review EN and PT translations
5. **Testing:** Verify all strings render correctly in all three languages
6. **Maintenance:** New features add all three languages simultaneously
